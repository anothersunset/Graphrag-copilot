## 3. 验证目标（验收红线）

| 指标 | 目标 | 备注 |
|---|---|---|
| Context Precision | ≥ 0.80 | RAGAS 标准 |
| Faithfulness | ≥ 0.85 | RAGAS 标准 |
| Recall@5 | ≥ 0.85 | 自有题库 |
| Multi-hop 准确率 | 较 v1 提升 ≥ 15 pp | v1 baseline 由 W1 D5 测出 |
| 幻觉率 | ≤ 10% | DeepEval |
| CRAG 修复率 | ≥ 70% | 中分数被改写后命中率 |
| Trace Completeness | = 1.00 | 所有调用必须落 Langfuse |
| Tool Call Necessity | ∈ [0.9, 1.1] | Auditor 计算（DSPy） |
| 审计覆盖率 | = 1.00 | 100% 请求经 Auditor |
| Test coverage | ≥ 70% | pytest + coverage |

## 4. CRAG 阈值（Evaluator 节点决策表）

| 置信度 | 行动 | 上限 |
|---|---|---|
| ≥ 0.7 | use（直接进入 Reasoner） | — |
| 0.3 ~ 0.7 | rewrite（查询改写 → 重试检索） | 2 次 |
| < 0.3 | fallback（web search / 主动拒答 / 标注"信息不足"） | — |

## 5. 安全护栏（高风险操作清单）

以下操作 **AI 不得自主执行**，必须在 Notion 主控页 @ 项目作者获得显式批准：

- `git push --force` / `--force-with-lease`
- 删除任何 GitHub 仓库
- 发布任何 GitHub Release
- `docker system prune` 或任何 `--volumes` 数据删除
- 任何 `rm -rf` 操作

## 6. 范围外（明确不做）

为防止范围爆炸，以下功能 **v3.1 内不做**，留待 v3.2+：

- ❌ 多租户 / RBAC / ACL（仅 X-API-Key + 限流）
- ❌ OAuth / mTLS / SSO
- ❌ Prometheus / OpenTelemetry exporter（用 Langfuse 即可）
- ❌ 大规模分布式（Qdrant 单机即可，不上 sharding）
- ❌ 移动端
- ❌ 多模态生成（仅检索/理解多模态，生成仍是文本）
